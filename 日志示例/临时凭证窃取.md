# CloudTrail 日志示例：EC2 Instance Profile 临时凭据窃取，攻击者外部调用

> **场景描述**：攻击者入侵了运行中的 EC2 实例（或通过 SSRF 漏洞访问了实例元数据服务），从 `http://169.254.169.254/latest/meta-data/iam/security-credentials/{role-name}` 窃取了 Instance Profile 的临时凭据（AK + SK + SessionToken），随后在自己的机器上用这组凭据调用 `GetCallerIdentity` 验证有效性。  
> **凭据类型**：IAM Role 临时凭据（`userIdentity.type = AssumedRole`，`accessKeyId` 前缀 `ASIA`）  
> **核心异常**：`sourceIPAddress` 是攻击者的外部 IP，而非 EC2 实例的内网或弹性 IP

## 字段说明

| 字段 | 值 | 说明 |
|---|---|---|
| `userIdentity.type` | `AssumedRole` | 临时凭据标识，区别于 IAM User 的 `IAMUser` |
| `userIdentity.accessKeyId` | `ASIAIOSFODNN7EXAMPLE` | 前缀 `ASIA` 是 STS 临时凭据的硬标识，对应凭据包含 AK + SK + SessionToken 三元组 |
| `userIdentity.principalId` | `AROAI3FHNJU4TPFNC7KGF:i-0a1b2c3d4e5f67890` | 冒号后半段 `i-0a1b2c3d4e5f67890` 是 Session Name，Instance Profile 自动以**实例 ID** 作为 Session Name |
| `userIdentity.arn` | `assumed-role/ec2-app-server-role/i-0a1b2c3d4e5f67890` | ARN 中的 Session Name 同样是实例 ID，可据此反查是哪台 EC2 的凭据被盗用 |
| `sessionContext.sessionIssuer.arn` | `role/ec2-app-server-role` | 签发临时凭据的原始 Role，是溯源的关键字段 |
| `sessionContext.ec2RoleDelivery` | `1.0` | **EC2 Instance Profile 专属字段**，标识凭据通过 IMDS 下发，其他途径（如 AssumeRole）不含此字段 |
| `sessionContext.attributes.creationDate` | `2025-11-14T01:08:43Z` | 凭据创建时间（EC2 启动或上次轮换时），与本次调用时间差约 **2h43m**——凭据并非刚刚颁发 |
| `sourceIPAddress` | `45.142.212.18` | **核心异常**：EC2 实例的正常调用来源应为实例内网 IP 或弹性 IP（如 `10.0.3.47`），此处是境外 VPS |
| `userAgent` | `aws-cli/2.17.23 ...` | **辅助异常**：EC2 上运行的应用通常使用 SDK（如 `aws-sdk-java`），裸 `aws-cli` 调用不符合正常应用行为 |

---

## 核心检测逻辑

**EC2 Instance Profile 凭据被盗的判断依据**，来自以下两个字段的组合：

`sessionContext.ec2RoleDelivery = "1.0"` 确认凭据来自 IMDS，说明这本是一张只应在 EC2 实例内部使用的凭据。

`sourceIPAddress` 不在该 EC2 实例的已知 IP 范围内，说明凭据已经被带出实例在外部使用。

这两个条件同时成立，即可高置信度判断 IMDS 凭据窃取。若只有前者没有后者（即 IP 正常），是合法的实例内部调用。

## 检测规则参考

```sql
-- CloudTrail Lake：检测 Instance Profile 凭据在 EC2 实例 IP 之外被使用
SELECT
  userIdentity.arn,
  userIdentity.accessKeyId,
  sourceIPAddress,
  userAgent,
  eventTime,
  eventName
FROM cloudtrail_logs
WHERE
  userIdentity.type = 'AssumedRole'
  AND json_extract_scalar(userIdentity, '$.sessionContext.ec2RoleDelivery') = '1.0'
  AND sourceIPAddress NOT IN (
    SELECT private_ip FROM ec2_inventory   -- 已知 EC2 实例 IP 清单
  )
  AND sourceIPAddress NOT LIKE '10.%'
  AND sourceIPAddress NOT LIKE '172.16.%'
  AND sourceIPAddress NOT LIKE '192.168.%'
ORDER BY eventTime DESC
```

## 与 IAM User 日志的关键差异

| 对比项 | IAM User（上一场景） | EC2 Instance Profile（本场景） |
|---|---|---|
| `userIdentity.type` | `IAMUser` | `AssumedRole` |
| `accessKeyId` 前缀 | `AKIA`（长期凭据） | `ASIA`（临时凭据） |
| `sessionContext` | 无 | **有**，含 `sessionIssuer` 和 `ec2RoleDelivery` |
| 凭据有效期 | 永久（手动吊销前） | 默认 **1 小时**，到期自动失效 |
| 溯源字段 | `userName` | `sessionContext.sessionIssuer.arn` + Session Name 中的实例 ID |
| 核心异常判断 | `sourceIPAddress` 不在白名单 | `ec2RoleDelivery = "1.0"` 且 `sourceIPAddress` 不是 EC2 实例 IP |

*日志结构基于 CloudTrail `eventVersion 1.09`，字段值为模拟生成，仅供检测规则测试使用。*
