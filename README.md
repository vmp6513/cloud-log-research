# 云安全日志调研项目

> 云平台日志体系调研、安全告警采集与归一化建模

## 项目概述

本项目针对主流云平台（AWS、Azure、Microsoft Entra ID）的日志体系进行系统性调研，形成可供安全运营团队使用的日志分析文档。调研内容包括：

- **日志分类与能力边界**：各云平台日志类型、覆盖范围、功能限制
- **采集方案设计**：多种低成本、低延迟的日志采集方案选型
- **归一化建模**：跨平台日志统一模型，支持告警研判与 LLM 分析


## 核心内容

### 1. 调研报告

详细分析各云平台的审计日志体系：

| 平台 | 核心审计日志 | 覆盖范围 |
|------|-------------|----------|
| AWS | CloudTrail | 控制面操作、IAM 事件 |
| Azure | Activity Log / Resource Logs | 订阅级操作、资源访问 |
| Entra ID | SignInLogs / AuditLogs | 身份认证、目录操作 |

### 2. 采集方案

提供三套采集方案供选型：

- **方案一（主流方案）**：存储层 + 消息队列，完整可靠
- **方案二（轻量方案）**：事件驱动，低成本过滤
- **方案三（最简方案）**：定时轮询，配置最简

### 3. 归一化模型

统一日志模型设计，支持跨平台关联分析：

- **Actor 统一主体**：区分 human / service_account / machine_identity / cloud_service
- **Action 动词化**：create / read / update / delete / auth / assume
- **攻击链追溯**：session_chain、on_behalf_of 字段
- **LLM 友好**：action.summary 自然语言摘要

### 4. 日志示例

典型攻击场景的完整日志样本，包含触发方式与研判意义。
